# ICMF Governance

ICMF is an open cybersecurity framework focused on insider code manipulation risks.

## Governance Goals

- maintain methodological consistency
- protect framework credibility
- ensure evidence-based technique evolution
- keep public definitions vendor-neutral

## Versioning

ICMF uses semantic framework releases:

- Major: taxonomy or methodology changes
- Minor: new techniques or material refinements
- Patch: editorial corrections and clarity improvements

## Technique Contribution Principles

A new technique should include:

- technique ID
- category
- technical definition
- evidence expectations
- detection indicators
- related risks

## Review Principles

Techniques should be reviewed for:

- technical clarity
- evidence threshold
- false positive risk
- overlap with existing techniques
- neutrality of language

## Neutrality Statement

ICMF is a public framework. Commercial tools may implement ICMF, but the framework itself is intended to remain implementation-neutral.
